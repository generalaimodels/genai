# Button Component

A customizable button component with multiple variants, sizes, and premium animations.

<Callout type="info">
This component is part of our design system and follows accessibility best practices.
</Callout>

## Props Reference

<PropsTable>
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| variant | `'primary' \| 'secondary' \| 'outline'` | `'primary'` | Button style variant |
| size | `'sm' \| 'md' \| 'lg'` | `'md'` | Button size |
| disabled | `boolean` | `false` | Disabled state |
| onClick | `() => void` | - | Click handler function |
</PropsTable>

## Usage Examples

<Tabs>
  <Tab label="Basic Usage">
    ```jsx
    import { Button } from '@/components/Button'
    
    function App() {
      return <Button variant="primary">Click Me</Button>
    }
    ```
  </Tab>
  
  <Tab label="With Icons">
    ```jsx
    import { Button } from '@/components/Button'
    import { Icon } from '@/components/Icon'
    
    function App() {
      return (
        <Button variant="primary">
          <Icon name="download" />
          Download
        </Button>
      )
    }
    ```
  </Tab>
  
  <Tab label="Disabled State">

    ```jsx
        </Button>
      )
    }
    ```
  </Tab>

## Accessibility

<Callout type="warning">
Always provide an accessible label for icon-only buttons using the `aria-label` prop.
</Callout>

- Supports keyboard navigation (Enter and Space keys)
- Proper ARIA attributes for screen readers
- Focus indicators for keyboard users
- Color contrast ratio meets WCAG AA standards

## Performance

<Callout type="tip">
The Button component is tree-shakeable and adds only ~2KB to your bundle when gzipped.
</Callout>

## API Summary

The button component uses CSS-in-JS for styling with zero runtime overhead through static extraction.
